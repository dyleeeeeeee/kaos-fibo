About Fibonacci Compression

This system implements a novel approach to compressing numerical datasets using Fibonacci coding, a universal code that makes use of Fibonacci numbers. This project explores the application of this mathematical concept in web-based data compression systems.

Project Overview

The Fibonacci Compression System is a web-based application designed to compress large numerical datasets using Fibonacci coding. The system provides a modern, user-friendly interface for data compression while offering detailed metrics and comparative analysis with other compression methods.

Key Features

  • Lossless Compression: Ensures complete data integrity with no information loss
  • Fibonacci Coding: Based on Zeckendorf's theorem for optimal representation
  • Real-time Processing: Instant compression with detailed performance metrics
  • Comparative Analysis: Benchmarks against Huffman and LZW algorithms
  • Modern UI: Glassmorphic design with responsive layout
  • Comprehensive Metrics: Detailed statistics including size, speed, and efficiency

Fibonacci Coding Algorithm

Fibonacci coding is a universal code which encodes positive integers into binary code words. It is based on the Fibonacci sequence and uses Zeckendorf's representation.

How It Works

Each positive integer can be represented uniquely as a sum of non-consecutive Fibonacci numbers (Zeckendorf's theorem). The Fibonacci code for an integer is the binary sequence representing which Fibonacci numbers are included in this sum, followed by an additional '1' as a terminator.

Key Properties

  • Prefix-free: No codeword is a prefix of another, enabling unambiguous decoding
  • Self-synchronizing: The '11' terminator allows easy boundary detection
  • Efficient for small numbers: Shorter codes for smaller integers
  • No consecutive 1s: Except for the terminating '11', adhering to Zeckendorf's theorem

Literature Review

Data compression has been a fundamental area of computer science research for decades. The field encompasses various approaches, from statistical methods to dictionary-based techniques.

Historical Context

Huffman Coding (1952): David Huffman developed optimal prefix codes based on character frequency. While highly effective for text compression, it requires frequency analysis and doesn't naturally handle numerical data.

LZW Algorithm (1984): Lempel-Ziv-Welch compression builds a dictionary of patterns and is widely used in GIF images and UNIX compress. It excels at finding repeated patterns but may be less efficient for numerical datasets.

Fibonacci Coding (1960s): Though the concept dates back further, modern applications of Fibonacci coding emerged in the 1960s. It represents a unique approach based on number theory rather than statistical or pattern-based methods.

System Architecture

The Fibonacci Compression System follows a three-tier architecture:

Presentation Layer (Frontend)

  • HTML5, CSS3 with glassmorphic design
  • Vanilla JavaScript for dynamic interactions
  • Responsive design for all device sizes
  • Real-time data visualization

Application Layer (Backend)

  • Python Flask RESTful API
  • Fibonacci coding algorithm implementation
  • Comparative analysis with Huffman and LZW
  • Performance metrics collection

Data Layer (Database)

  • MongoDB Atlas cloud database
  • Compression history logging
  • Metrics storage and retrieval
  • Scalable document-based storage

Evaluation Metrics

The system evaluates compression performance using multiple criteria:

Size Metrics

  • Compression Ratio: Original size / Compressed size
  • Size Reduction: Percentage of space saved
  • Bytes Saved: Absolute reduction in storage

Performance Metrics

  • Compression Time: Time taken to compress data
  • Throughput: Data processed per unit time
  • Resource Usage: CPU and memory consumption

Comparative Metrics

  • vs Huffman: Comparison with statistical coding
  • vs LZW: Comparison with dictionary-based coding
  • Best Method: Identifies optimal algorithm for given data

Scope & Limitations

Scope

  • Prototype system not intended for production deployment
